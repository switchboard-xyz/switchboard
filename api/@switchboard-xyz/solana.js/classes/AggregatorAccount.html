Class AggregatorAccount

Account type holding a data feed's update configuration, job accounts, and its current result.

Data: AggregatorAccountData

Result: SwitchboardDecimal

HistoryBuffer?: Array<AggregatorHistoryRow>

An aggregator account belongs to a single QueueAccount but can later be transferred by the aggregator's authority. In order for an OracleAccount to respond to an aggregator's update request, the aggregator must initialize a PermissionAccount and LeaseAccount. These will need to be recreated when transferring queues.

Optionally, An aggregator can be pushed onto a CrankAccount in order to be updated

Optionally, an aggregator can add a history buffer to store the last N historical samples along with their update timestamp.
